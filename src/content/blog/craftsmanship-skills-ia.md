Les compétences du crafts — TDD, clean code, DDD, SOLID, etc. — **n'ont jamais eu autant de valeur**.

Parce qu'on les enseigne à nos outils.

## L'IA amplifie ce que vous savez

Quand vous demandez à un agent IA de coder, il produit ce que vous lui demandez. Si vous maîtrisez les principes du craftsmanship, vous obtenez du code propre, testé, bien architecturé. Si vous ne les maîtrisez pas, vous obtenez du spaghetti — mais plus vite.

L'IA est un **multiplicateur de compétences**, pas un substitut.

## Plus d'excuse

Avant, on entendait :

> "On n'a pas le temps de bien faire."

Cet argument ne tient plus. Claude, Copilot, et les autres agents codent vite **et** proprement — si on leur donne les bonnes instructions.

Le développeur qui maîtrise :
- **TDD** : obtient des tests pertinents générés rapidement
- **Clean Code** : obtient du code lisible dès la première itération
- **DDD** : obtient un modèle métier cohérent
- **SOLID** : obtient une architecture flexible et maintenable

## Un choix, pas une contrainte

Faire du code dégueulasse devient **un choix**, pas une contrainte de temps.

L'IA supprime l'excuse du "pas le temps". Elle rend la production de code propre aussi rapide que la production de code sale.

## Le craftsmanship comme différenciateur

Le développeur qui investit dans ces compétences :

1. **Guide mieux ses outils** : des prompts précis, des attentes claires
2. **Détecte les erreurs** : il sait reconnaître du mauvais code, même généré par l'IA
3. **Maintient la cohérence** : il assure que le code généré s'intègre dans l'architecture existante

Le craftsmanship n'est pas menacé par l'IA. Il est **amplifié** par elle.
